# G3-F15 NOESY 峰归属方法与实验结果分析

本文档说明当前 `assign_noesy_g3f15.py` 对 G3-F15 序列 `TCGGGAAGGGAGGG` 进行 NOESY-only 峰归属时采用的推理规则、计算流程、证据权重和实验结果解释。分析对象为 `noesy_g3f15x7.xpk`，结果文件来自 `results/assignment_table.csv`、`results/assignment_candidates.json` 和 `results/assignment_summary.md`。

## 1. 方法定位

当前方法的目标是从 2D NOESY 峰表中给出一版保守的初步序列归属，而不是直接给出最终三维结构或完全无歧义的全质子化学位移表。

该方法只使用 NOESY 数据，因此主要依赖空间近邻关系：

- 芳香质子 H6/H8 与糖环 H1' 的同残基和相邻残基 NOE。
- T 残基甲基与 T-H6 的强 NOE 锚点。
- G-quadruplex 中鸟嘌呤 imino H1 的低场信号及其与芳香区的 NOE。
- NOE 强度的半定量距离信息。

TOCSY 可以帮助确认同一残基内部糖环自旋系统，但当前数据没有 TOCSY，因此脚本不会把 H1'-H2'/H2'' 等关系作为硬约束。对于多解或重叠峰，脚本采用保守策略，将结果标记为 `ambiguous` 或 `medium`，而不是强行输出唯一答案。

## 2. 使用的主要推理规则

### 2.1 NOE 强度与距离规则

NOESY 交叉峰强度与质子间距离通常可近似看作反比于距离的六次方，即 `I ~ r^-6`。因此脚本把强峰视为较强空间近邻证据，把弱峰视为较弱证据。

本项目没有把峰强度转换为绝对距离，而是按相对强弱使用：

| 强度类别 | 当前脚本阈值 | 解释 |
| --- | ---: | --- |
| `strong` | `int >= 20` | 高可信空间近邻，常见于紧密堆积或明显同残基/相邻残基 NOE |
| `medium` | `6 <= int < 20` | 可用证据，但需要结合 sequential walk |
| `weak` | `2 <= int < 6` | 支持性证据，单独不足以定论 |
| `very_weak` | `int < 2` | 极弱证据，通常标记风险 |

### 2.2 化学位移分区规则

脚本先按化学位移区间把峰分成不同证据来源。

| 证据类型 | 化学位移区域 | 用途 |
| --- | --- | --- |
| 芳香-H1' 区 | `5.0-6.5 ppm` 对 `7.0-8.6 ppm` | H6/H8-H1' 的同残基峰和 sequential walk |
| T methyl-base 区 | `1.45-2.05 ppm` 对 `7.0-8.6 ppm` | T1 甲基-H6 锚点 |
| G imino-aromatic 区 | `10.5-12.5 ppm` 对 `7.0-8.6 ppm` | G-quadruplex imino 辅助验证 |

这里的方向不依赖 `.xpk` 中 Hx/Hy 谁在横轴或纵轴。脚本会自动把低场/高场组合统一为 `low` 和 `high`，例如 H1' 总是作为 `low`，芳香 H6/H8 总是作为 `high`。

### 2.3 峰聚类规则

同一个化学位移在峰表中会因对称峰、重复拾峰或微小误差出现多个近似值。脚本使用默认 `0.03 ppm` 容差进行聚类。

本次数据检测到的主要聚类如下：

- H1'/近 H1' 区：`5.011, 5.080, 5.132, 5.361, 5.422, 5.464, 5.543, 5.646, 5.693, 5.782, 5.836, 5.884, 5.956, 6.010, 6.074, 6.127, 6.194, 6.265, 6.317, 6.412 ppm`
- 芳香碱基区：`7.301, 7.484, 7.576, 7.611, 7.683, 7.756, 7.830, 7.916, 7.974, 8.033, 8.067 ppm`
- imino 区：`10.901, 11.041, 11.173, 11.229, 12.066, 12.172, 12.308, 12.349, 12.416 ppm`

聚类后的峰被合并为同一个候选证据，报告中的 `peak_ids` 会列出该证据对应的原始峰号。

### 2.4 T1 锚点规则

序列 `TCGGGAAGGGAGGG` 中只有一个 T，即 T1。因此 T 甲基是 5' 端定位的重要锚点。

T methyl 的经验规则：

- T-CH3 通常位于约 `1.5-2.0 ppm`。
- T-CH3 与同残基 T-H6 常有明显 NOE。
- T-H6 一般位于芳香区，脚本使用 `7.0-8.6 ppm` 作为搜索范围。
- T1 的 H1' 仍需要被芳香-H1' 区和后续 C2/G3 walking 共同支持，不能只看甲基最强峰。

本次数据中最强 T methyl-base 候选为：

| methyl ppm | base ppm | 峰号 | 强度 |
| ---: | ---: | --- | ---: |
| 1.7523 | 7.4840 | 92 | 43.317 |
| 1.7523 | 7.5758 | 93;97 | 11.024 |
| 1.7523 | 7.3007 | 89;90;514 | 8.436 |
| 1.7523 | 7.6826 | 94;992 | 6.711 |
| 1.7523 | 7.7557 | 99 | 6.649 |

最终脚本没有单纯选择最强 methyl 峰 `7.4840 ppm`，而是结合 H1' 合理性和 sequential walk，给出 T1：`H6 = 7.5758 ppm`，`H1' = 5.7824 ppm`。这说明当前算法优先选择“能接入全序列路径”的锚点，而不是只追求局部最强峰。

### 2.5 H6/H8-H1' 同残基规则

核酸 NOESY 指纹区通常存在同残基相关：

`H6/H8(i) -> H1'(i)`

脚本把每一个芳香-H1' 峰簇作为一个候选状态：

- `base_ppm` 表示候选 H6/H8 化学位移。
- `h1prime_ppm` 表示候选 H1' 化学位移。
- `intra_peak_ids` 表示支持该同残基候选的峰号。
- `intra_intensity` 表示该候选中最强峰强度。

同残基峰越强，候选状态得分越高；如果同残基峰非常弱，即使路径上能连通，也会降级为 `medium` 或 `ambiguous`。

### 2.6 顺序步行规则

序列归属的核心是 sequential walking。对于相邻残基，脚本寻找：

`H6/H8(i+1) -> H1'(i)`

即下一个残基的芳香质子与前一个残基 H1' 的 NOE。这样可以从 T1 沿序列走到 C2、G3、G4 等。

脚本的顺序连接规则包括：

- 当前残基的芳香峰必须能与上一残基的 H1' 聚类形成 NOE。
- 顺序峰强度越高，路径得分越高。
- 如果顺序证据复用了同一个内峰，脚本会扣分。
- 如果相邻残基复用同一个 H1' 聚类，脚本会扣分并在结果中记录。
- 如果相邻不同碱基类型复用同一个芳香峰聚类，脚本会扣分。

这些惩罚是为了避免算法只沿着几个强重叠峰循环，而忽略真实序列中每个残基应有相对独立的化学位移状态。

### 2.7 碱基类型化学位移窗口规则

脚本对 T、C、G、A 使用不同的经验化学位移窗口进行加权。

简化理解如下：

- T：偏好 `base 7.25-7.85 ppm`，`H1' 5.45-5.90 ppm`。
- C：偏好 `base 7.25-7.90 ppm`，`H1' 5.20-5.90 ppm`。
- G：偏好 `base 7.45-8.20 ppm`，`H1' 5.20-6.30 ppm`。
- A：偏好 `base 7.30-8.35 ppm`，`H1' 5.20-6.35 ppm`。

这些窗口不是硬性排除规则，而是得分权重。因为 G-quadruplex、末端堆积、局部环区构象和峰重叠都可能导致化学位移偏离常规区间。

### 2.8 G imino 验证规则

G-quadruplex 中参与 Hoogsteen 氢键的 G imino H1 往往出现在低场区，常见范围约 `10.5-12.5 ppm`。脚本会在这个区间寻找 imino-aromatic NOE，并把它作为 G 残基的辅助证据。

本次数据中 imino 相关聚类集中在：

- `10.901 ppm`
- `11.041 ppm`
- `11.173 ppm`
- `11.229 ppm`
- `12.066-12.416 ppm` 的若干弱/分散聚类

计算结果中多个 G 被关联到 `11.1727 ppm` imino 聚类，说明 imino 区确实存在 G-quadruplex 相关信号，但单靠当前 2D NOESY 峰表仍难以把每个 G 的 imino H1 完全拆分到唯一残基。

### 2.9 路径搜索和保守置信度规则

脚本使用 beam search 搜索整条序列的候选路径。每个残基状态由以下证据共同打分：

- 同残基 H6/H8-H1' 峰强度。
- 与前一残基的 sequential peak 强度。
- 碱基类型化学位移窗口。
- T methyl-H6 锚点证据。
- G imino-aromatic 证据。
- 峰复用、H1' 聚类复用、芳香聚类复用等扣分项。

输出置信度分为：

- `high`：路径支持强，候选路径中稳定出现。
- `medium`：有可用证据，但内峰或局部证据偏弱。
- `ambiguous`：候选路径之间分数接近、posterior 偏低或存在峰复用风险。
- `low_confidence`：证据很弱或缺关键连接。

本次最佳路径分数为 `220.392`，最佳路径与第二路径只差 `0.211`。这个差距很小，说明数据中存在明显多解，必须保留歧义标记。

## 3. 实验数据概况

脚本从 `noesy_g3f15x7.xpk` 中解析到：

| 项目 | 数量 |
| --- | ---: |
| 总峰数 | 1245 |
| 芳香-H1' 区峰 | 88 |
| T methyl-base 区峰 | 32 |
| imino-aromatic 区峰 | 33 |

整体上，数据具备进行 NOESY-only walking 的基础：

- 芳香-H1' 区峰数量足够支撑初步 sequential walk。
- `1.75 ppm` 附近存在 T methyl 相关峰，可用于 T1 锚定。
- `10.9-11.3 ppm` 附近存在明显 imino 相关峰，符合 G-quadruplex 的低场 imino 特征。

但数据也有明显限制：

- 多个残基共享或接近相同 H1' 聚类。
- 芳香区 `7.68-7.83 ppm` 附近峰重叠较多。
- imino 信号可以证明 G-rich 折叠特征，但不能完全一一拆解每个 G 的 imino 归属。
- 缺少 TOCSY，糖环内部自旋系统不能直接被独立确认。

## 5. 结果解释

### 5.1 T1 锚点可靠

T1 被归属为 `H6 = 7.5758 ppm`、`H1' = 5.7824 ppm`。该结果同时得到：

- 芳香-H1' 同残基峰 `42;214` 支持。
- T methyl-base 峰 `93;97` 支持，methyl ppm 约 `1.7523 ppm`。
- 后续 C2 的 sequential walk 可以从 T1 的 H1' 继续连接。

虽然 methyl-base 区最强峰是 `1.7523/7.4840 ppm` 的峰 92，但它没有在当前全路径中给出更好的整体连接。因此当前报告把 T1 选在 `7.5758/5.7824 ppm`，这是“全局路径最优”的结果。

### 5.2 5' 端 T1-C2-G3 有可走通路径，但 C2/G3 保留歧义

C2 的结果为 `base = 7.7557 ppm`、`H1' = 5.6465 ppm`，其顺序连接峰 `41;233` 强度较好。但 C2 的 posterior 只有 `0.49`，说明其它路径竞争接近。

G3 的结果为 `base = 7.6826 ppm`、`H1' = 6.1940 ppm`，顺序连接峰 `51;1012;1225` 支持从 C2 到 G3。G3 也有 imino 相关证据，但 posterior 为 `0.40`，因此仍应人工复核。

### 5.3 G4、G14 是较稳定节点

G4 和 G14 被标为 `high`：

- G4：`base = 7.7557 ppm`，`H1' = 6.0739 ppm`，同残基峰 `5;922` 很强，顺序连接峰 `44;920` 也强。
- G14：`base = 7.6826 ppm`，`H1' = 5.5430 ppm`，同残基峰 `1011` 和顺序连接峰 `50;543` 都具有可用强度。

这两个点可以作为后续人工在谱图中回查的稳定节点。

### 5.4 G5 和 A6 可用但证据偏弱

G5 和 A6 被标为 `medium`：

- G5 同残基峰强度只有 `3.054`，属于 weak。
- A6 同残基峰强度只有 `3.381`，也属于 weak。

二者路径连接强，但同残基内峰偏弱，因此不能标为 high。建议在原谱中复查这两个点是否受重叠、拾峰阈值或相位/基线影响。

### 5.5 G8-G13 区域是主要不确定区域

G8、G9、G10、A11、G12、G13 均被标记为 `ambiguous`。主要原因包括：

- G8 和 G12 的 H1' 聚类与其它残基复用。
- G9、G10、A11 的 top-path posterior 很低，说明候选路径竞争激烈。
- G13 的同残基峰 `1046` 强度只有 `1.783`，属于 very weak。
- G-rich 核心区本身容易出现芳香峰重叠和 imino 信号重叠。

这一段不应直接作为最终唯一归属，需要结合人工谱图检查、更多 NOESY 混合时间数据、TOCSY 或后续结构计算约束继续确认。

### 5.6 imino 区支持 G-quadruplex，但个别 G-H1 归属仍不唯一

多个 G 残基被关联到 `11.1727 ppm` imino 聚类。这个现象支持存在 G-quadruplex 相关的 G imino 信号，但也提示 imino 区存在峰重叠。

因此当前结论应表述为：

- 数据支持 G-rich 序列形成具有 G-quadruplex 特征的折叠状态。
- `10.9-11.3 ppm` 的 imino 区信号是重要支持证据。
- 但当前脚本不能仅凭这份 NOESY 峰表唯一确定所有 G tetrad 层内对应关系。

## 6. 当前结论

当前 NOESY-only 计算完成了从原始 `.xpk` 峰表到序列归属候选表的自动化推理。结果中 T1 锚点较可靠，G4 和 G14 是较稳定的 walking 节点，G5/A6 可作为中等可信候选。G8-G13 区域因峰重叠和多路径竞争，应保留歧义。

这份结果适合作为下一步人工归属和结构计算前的候选约束整理基础，但不建议直接作为最终 CYANA/XPLOR-NIH 输入的唯一化学位移表。后续如果加入 TOCSY、不同 mixing time 的 NOESY，或人工确认的已知峰，可以把这些信息作为额外先验加入脚本，提高 G-rich 核心区的唯一性。
